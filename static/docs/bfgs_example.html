1. Introduction ¶

The BFGS (Broyden-Fletch-Goldfarb-Shanno) class of algorithms minimizes a given objective function by iteratively updating an initial parameter estimate $x^{(0)}$ with a scaled approximation of the inverse Hessian matrix $B^{(k)}$ (computed at every iteration) multipliedby the gradient $\nabla f^{(k)}$. Intuitively, each update improves upon the previous estimate $x^{(k)}$ by taking a step opposite the direction that of the function's growth (and therefore, in the direction of the objective function's minimum). For details of the algorithm, see Ch. 6, 7 of Nocedal and Wright (2006)[[1]](#1).

This notebook implements the BFGS and L-BFGS (limited memory BFGS) algorithms for solving nonlinear optimization problems. It tests these methods on the nonlinear least squares regression problem, fitting an exponential class function to two strongly-blended Gaussians on a decaying exponential baseline plus normally distributed zero-mean noise with variance = 6.25[[2]](#2).

2. Problem Setup ¶

In the following section, we define several functions to set up the nonlinear least squares problem.

2.1 Hypothesis Function ¶

This function returns the output of the predictive model for which we are attempting to find the best set of parameters, described by the exponential class:

$h_\beta(x) = \beta_1 e^{-\beta_2 x} + \beta_3 e^{-(x-\beta_4)^{2}/\beta_5^{2}} + \beta_6 e^{-(x-\beta_7)^{2}/\beta_8^{2}}$

In [4]:
def eval_hyp_fn(data, beta):
    """Evaluates the output of the exponential class
    hypothesis function described by

    h_b(x) = b1*exp( -b2*x ) + 
                b3*exp( -(x-b4)**2 / b5**2 ) + 
                b6*exp( -(x-b7)**2 / b8**2 )
    """
    x = data[0]
    h = np.zeros(x.shape)
    for i in range(len(x)):
        h[i][0] = (beta[0] * np.exp(-beta[1] * x[i][0]) + \
                    beta[2] * np.exp(-(x[i][0] - beta[3])**2 / beta[4]**2) + \
                    beta[5] * np.exp(-(x[i][0] - beta[6])**2 / beta[7]**2))[0]
    return h

2.2 Objective Function ¶

This function returns the output of the objective function we will attempt to minimize using BFGS. It is described by the sum of squared errors between the observed response variable and the prediction generated by the hypothesis function:

$f_\beta(x, y) = \vert\vert y - h_\beta(x) \vert\vert_2^{2}$

In [5]:
def eval_obj_fn(data, beta):
    """Evaluates the objective function, defined by
    the square of the LSE of the hypothesis function
    """
    x, y = data[0], data[1]
    h = np.zeros(x.shape)
    for i in range(len(x)):
        h[i][0] = (beta[0] * np.exp(-beta[1] * x[i][0]) + \
                    beta[2] * np.exp(-(x[i][0] - beta[3])**2 / beta[4]**2) + \
                    beta[5] * np.exp(-(x[i][0] - beta[6])**2 / beta[7]**2))[0]
    return np.sum(np.square(y - h))

2.3 Gradient of the Objective Function ¶

This function evaluates $\nabla f_\beta(x, y)$, the gradient of the objective function wrt. the parameters for which we are attempting to minimize.

In [6]:
def eval_grad_obj(data, param_vals):
    """Returns the gradient f of the LSE residuals 
    of the exponential class function wrt each parameter b_i
    """
    x = torch.tensor(data[0])
    y = torch.tensor(data[1])
    beta = torch.tensor(param_vals, requires_grad=True)
    f = beta[0] * torch.exp(-beta[1] * x) + \
         beta[2] * torch.exp(-(x - beta[3])**2 / beta[4]**2) + \
         beta[5] * torch.exp(-(x - beta[6])**2 / beta[7]**2)
    r = y - f
    r_lse = torch.sum(r**2)
    r_lse.backward()
    grads = beta.grad.numpy()
    return grads

2.4 Initial Estimate of the Hessian Matrix ¶

This function initializes the approximate Hessian Matrix $B^{(0)}$ as $J^{T}J$, where $J$ is the Jacobian matrix of the residual loss function $r_\beta(x, y) = y - h_\beta(x)$. For further details, see Ch. 10 of Nocedal and Wright (2006)[[1]](#1).

In [7]:
def init_H(data, param_vals):
    """Evaluates the approximate Hessian of the objective function
    with the truncated Taylor polynomial using J^T * J
    """
    x = torch.tensor(data[0])
    y = torch.tensor(data[1])
    beta = torch.tensor(param_vals, requires_grad=True)
    jacobian = np.zeros((len(x), len(param_vals)))
    for i in range(len(x)):
        f_i = beta[0] * torch.exp(-beta[1] * x[i]) + \
         beta[2] * torch.exp(-(x[i] - beta[3])**2 / beta[4]**2) + \
         beta[5] * torch.exp(-(x[i] - beta[6])**2 / beta[7]**2)
        r = y[i] - f_i
        r.backward()
        grad_r = beta.grad.numpy()
        jacobian[i] = grad_r[:,0]
    return np.matmul(np.transpose(jacobian), jacobian)

3.2.1 L-BFGS ¶

L-BFGS approximates $B^{(k)-1} \nabla f^{k}$ at each iteration with stored curvature information from only the last m updates, thereby reducing runtime as the full Hessian is not computed at each iteration.

4.3 Discussion ¶

As L-BFGS does not have access to the full Hessian matrix, the search direction $B^{(k) -1} \nabla f^{(k)}$ varies more frequently between iterations and is thus more sensitive to the behavior of the objective function in the last $m$ iterations. Varying the initial choice of the step-size $\alpha$ tends to yield more significant differences in the resulting minimized function for L-BFGS than in BFGS, particularly for small values of m. Although these factors prevent L-BFGS from always finding a global minimum in practice, it nonetheless manages reasonable performance similar to that of BFGS in early iterations, and depending on the behavior of the objective function in certain windows it may also converge more quickly than BFGS.

A comparison between the BFGS class of algorithms and the batch gradient descent algorithm demonstrates that second order hill climbing techniques are more expressive than first order hill climbing techniques. Indeed, on the nonlinear least squares problem defined in 2. Problem Setup the BFGS class of algorithms produces a better initialization and convergences significantly more quickly batch gradient descent.
